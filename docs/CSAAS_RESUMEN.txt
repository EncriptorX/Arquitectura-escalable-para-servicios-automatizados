================================================================================
FLUJO DE PROVISIONAMIENTO AUTOMÁTICO
================================================================================

1. Usuario completa formulario → Validación frontend
2. POST /api/csaas-provision → Backend recibe datos
3. Genera subdominio único → cliente-abc12345.suncarsrl.com
4. Verifica CNAME target → customers.suncarsrl.com (crea si no existe)
5. Crea CNAME proxied → subdominio → target
6. Crea Custom Hostname → SSL DV HTTP
7. Polling (5s x 60) → Espera status "active"
8. Aplica 7 reglas de seguridad → WAF, HTTPS, etc.
9. Almacena en memoria → Dict Python
10. Devuelve resultado → URL protegida + logs
11. Frontend muestra éxito → Página de resultados

Tiempo total: 1-5 minutos

CÓMO USAR EL SISTEMA
================================================================================

DESDE EL FRONTEND:
1. Accede a la aplicación web
2. Clic en "Protección CSaaS" o "CSaaS" en el header
3. Completa el formulario:
   - Nombre del cliente
   - ID del cliente (opcional)
   - URLs a proteger (FQDN)
4. Clic en "Provisionar Protección CSaaS"
5. Espera 1-5 minutos
6. Recibe URL protegida: https://cliente-abc123.suncarsrl.com

DESDE LA API:
curl -X POST https://tu-dominio.com/api/csaas-provision \
  -H "Content-Type: application/json" \
  -d '{
    "client_name": "Acme Corporation",
    "client_id": "CLI-123",
    "urls": ["app.acme.com", "api.acme.com"]
  }'

LISTAR CLIENTES:
curl https://tu-dominio.com/api/csaas-list

================================================================================
REGLAS DE SEGURIDAD APLICADAS
================================================================================

Cada cliente provisionado recibe automáticamente:

1. SSL/TLS Automático - Certificado DV por HTTP
2. WAF Managed Rules - Firewall de aplicaciones web
3. HTTPS Redirect - Redirección automática
4. Security Level: High - Nivel de seguridad alto
5. Bot Fight Mode - Protección contra bots
6. Browser Integrity Check - Verificación de navegador
7. Rate Limiting - Limitación de tasa (firewall rules)
8. Challenge Passage - Tiempo de validez 30 min
